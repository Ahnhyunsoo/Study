절차지향 = 동사중심 위주
함수를 위주로 기능을 만들어감
객체지향 = 주어중심 위주
캐릭터 같은 주어를 중심으로 기능을 만들어감
객체 = 현실에 존재하는 사물을 추상화하여 프로그램코드로 옮겨놓은것
객체지향프로그래밍 = OOP(object oriented programming)
변수와 함수를 동시에 지닐수 있는 데이터 집합을 객체(인스턴스 + 클래스) 라고한다.
클래스 = 자료형(도구) 인스턴스 = 메모리
인스턴스화 = 사용할 때 메모리에 할당하고 사용하겠다.
클래스의 4개속성
은닉화 다형성 상속성 캡슐화 
OOP의 특성 -> 추상화를 하기위해 클래스가필요하고 클래스의 4대속성은 은닉화 다형성 상속성 캡슐화가 있다.
c++ = 객체지향프로그래밍 + 일반화 프로그래밍
제네릭 프로그래밍 = 일반화 프로그래밍 ( 코드의 재사용성을 위해 사용하는문법)
