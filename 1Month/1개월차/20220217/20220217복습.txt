상수 = 사회 구성원들이 정해놓고 약속한 변하지않는 숫자 ex) 10 , 'A'

리터럴 상수 = 10,'A' (정해놓은것)
심볼릭 상수 = 프로그램 코드에 의해서 문법적으로 상수화되어진 숫자를 의미 ex) define, const, enum

변수 = 코드의 흐름에 따라서 값이 변할 수 있는 숫자 또는 문자
프로그래밍시 숫자를 메모리 공간에 저장을 해둬야 하는데 이때 메모리 공간의 값은 코드의 흐름에 따라 변할 수 있다.

자료형 = 어떤 데이터를 얼마만큼의 메모리 공간을 저장할지 나누기위한 형태

기본자료형의 종류(원시 데이터형)


정수타입 = short(2byte / 16bit), int(4byte / 32bit), long(4byte / 32bit), longlong(8byte / 64bit)
short형 2byte / 16bit의 표현 가짓수 = 65536개 , -32768~32767
실수 타입 : float(4byte / 32 bit), double(8byte / 64bit), long double(8byte) 실수는 메모리를 나눌때 고정 소수점과 부동 소수점방식이 있다.
고정소수점 = ㅁ(부호비트)ㅁㅁㅁㅁ(정수부분) ㅁㅁㅁㅁㅁㅁ (소수점부분)
			        지수부		   가수부
부동소수점 = ㅁ(부호비트)ㅁㅁ(정수부분)ㅁㅁㅁㅁㅁㅁㅁㅁㅁ(소수점부분)

글자타입 = char(1byte / 8bit)

논리타입 = bool(1byte / 8bit) 참 또는 거짓 / true false / 1 0
참 거짓 표현은 0또는1로 할수있는데 1byte인 이유는 최소 저장단위가 1byte이기때문

정수타입 자료형 앞에는 signed가 생략되어 있다. 그래서 음수, 양수 모두 표기 및 저장이 가능하다.
만약 양수 타입의 값만 저장하고 싶을 경우 unsigned를 붙여서 의식적으로 양수를 저장하겠다고 알려줘야 한다.

실수타입 자료형 앞에는 표식을 붙일 수 없다.

일반적으로 게임 제작 시에는 float 변수를 많이 사용하고 사용 시점에 f를 붙여서 float형의 실수 값인 것을 알려주자. ex) fTest
3.14는 리터럴상수로 double형이지만 3.14f 를 붙혀서 float형태로 표현할수있다. 2.0을 2.f도 가능
단 double형의 경우 0.0까지 표현해줘야함

문자타입 자료형 = 아스키코드표와 연동되어 0~127번에 해당하는 문자 값을 사용할 수 있다.
아스키코드에 존재하는 단 한 글자만 저장할 수 있는 1byte 용량을 가지고 있다.

논리 자료형 = 참과 거짓을 표현하기 위판 자료형 , 1byte자료형
0이 거짓 1이 참이지만 0.000001도 참이다. 결국엔 값이 있기때문 보통 이렇게 사용하지는 않는다.
