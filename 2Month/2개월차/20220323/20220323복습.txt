서로 상호 참조하는경우 무한루프에 빠지기 때문에 전방선언을 사용한다.
전방선언은 헤더파일에서 class 자료형을 선언해주는 것이다.

전방 선언(자료형의 유무 판단만 할뿐 클래스 내의 변수와 함수에 대한 정보는 전혀 알 수가 없다)

1. 전방 선언 이후, 객체 타입은 멤버 변수로 선언이 불가하여 반드시 포인터 타입으로 선언해야 한다.
2. cpp파일에 헤더파일을 받드시 포함하여 자료형 또는 함수의 호출 권한을 부여할 수 있게 해야 한다.

c단계의 선언 및 초기화 = int a = 0;
c++단계의 선언 및 초기화 int a = 0; or int a(0);

explicit = 컴파일러가 묵시적 형변환을 할 수 없게 제한하는문법
Class c = 10;
이렇게 하면 클래스에 정수형을 대입하기 때문에 안되는데
컴파일러가 자동형변환을 해주기 때문에 정상적으로 작동한다.
explicit을 사용할 경우 묵시적형변환을 제한하기 때문에 오류가 난다.
컴파일러가 임의로 행동을 하면 안되기때문에 항상 제한을 해주는게 좋다. (소멸자는 절대 explicit를 사용하지않는다)
