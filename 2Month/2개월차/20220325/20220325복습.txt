복사 생성자가 호출되는 경우
1. 객체를 생성할 때 먼저만든 객체원본을 매개변수로 넣어서 객체를 생성하는 경우 (주로 이 경우에만 사용)
2. 함수의 매개변수로 객체를 넘길 때
3. 함수의 반환타입이 객체일 때

복사 생성자는 기본 생성자가 없으면 성립될 수 없다. (원본 객체를 넘겨야하는데 기본 생성자가 없으면 원본 객체를 생성할 수 없다.)
얕은 복사 : 디폴트 복사생성자를 이용할 때 매개변수가 포인터 타입이면 소멸자 호출시 원본 힙주소가 삭제되고 사본의 힙주소를 또 삭제하려하기 때문에 문제가 생긴다.
깊은 복사 : 위의 문제를 예방하기 위해서 복사생성자를 직접 재정의하여 사본또한 동적할당을 해주면 문제가 해결된다.

extern = 이런 자료형이 있다고만 알려주는 문법
전역 변수를 다른파일에서 쓰기 위해서 사용한다. 예를들어 main.cpp에서 전역변수로  int a = 10 을 선언하고 이 전역변수를 sub.cpp 에서 사용하고 싶으면
헤더파일에 extern int a;를 작성하고 sub.cpp파일에 헤더선언을 하고 sub.cpp파일에서 a를 직접 사용할 수 있다.

this = 자신의 주소를 저장하고 있는 상수포인터
객체는 생성시에 this를 가지며 모든 맴버변수에는 this가 생략되어 있다.
때문에 객체를 통해 같은 함수와 맴버변수를 호출 하더라도 this덕분에 해당하는 객체주소의 맴버변수가 호출된다.
